# Model-Fingerprint Adapter: Fingerprint Step

This step note is grounded in:

- `pipeline_adapter.py`
- `utils/pipeline.py`
- `run_clm.py`
- `inference.py`
- `create_fingerprint_mix.py`
- `README.md`

## 1. Purpose

Create a published-model directory that:

- behaves normally without the private adapter, and
- activates the fingerprint when the private adapter is loaded.

## 2. Required Inputs

Mandatory inputs for the current official Adapter route:

- base model name that exists in `configs/adapter.yaml`
- fingerprint dataset directory:

```text
dataset/llama_fingerprint_mix
```

- prompt template:

```text
barebone
```

The dataset is generated by:

```bash
python3 create_fingerprint_mix.py
```

## 3. Recommended Minimal Command

Mainline command:

```bash
python3 pipeline_adapter.py fingerprint --base_model mistralai/Mistral-7B-v0.1
```

Shorter smoke-style variant for this round:

```bash
python3 pipeline_adapter.py fingerprint --base_model mistralai/Mistral-7B-v0.1 epoch=1
```

Why this works without a new config file:

- `utils/pipeline.py` supports `key=value` runtime overrides.
- The key must already exist in `configs/adapter.yaml`.
- `epoch` already exists, so it can be reduced for a smoke run without modifying upstream config structure.

## 4. What the Pipeline Actually Executes

For Adapter mode, `pipeline_adapter.py` appends:

1. `accelerate launch ... run_clm.py ... --freeze_instruction_nonembedding --instruction_nonembedding_dim=<dim>`
2. `python inference.py <fingerprinted_dir> <data_path> publish_w_adapter ...`
3. `python inference.py <fingerprinted_dir> <data_path> publish --dont_load_adapter ...`
4. `python inference.py <base_model> <data_path> vanilla --dont_load_adapter ...`

The actual hyperparameters come from `configs/adapter.yaml`.

For `mistralai/Mistral-7B-v0.1` they are:

```yaml
epoch: 15
lr: "1e-2"
dim: 16
total_bsz: 8
data_name: mix
```

Because `total_bsz == 8`, `pipeline_adapter.py` uses:

- `per_device_train_batch_size=1`
- `gradient_accumulation_steps=1`

## 5. Output Directory and Expected Files

Default output directory for the recommended candidate:

```text
output_barebone_adapter/mistralai/Mistral-7B-v0.1/mix_epoch_15_lr_1e-2_bsz_8_d_16/
```

Smoke override with `epoch=1` changes that directory to:

```text
output_barebone_adapter/mistralai/Mistral-7B-v0.1/mix_epoch_1_lr_1e-2_bsz_8_d_16/
```

Expected important outputs:

- model weights and tokenizer files from `trainer.save_model()`
- `instruction_emb.pt`
- `publish_w_adapter.jsonl`
- `publish.jsonl`
- `vanilla.jsonl`

## 6. How to Validate the Step

Minimal validation after a completed run:

1. Confirm the directory exists.
2. Confirm `instruction_emb.pt` exists.
3. Confirm the three JSONL files exist.
4. Inspect `publish_w_adapter.jsonl`; per README, the first 10 rows should generate `"ハリネズミ"`.

Suggested checks:

```bash
ls output_barebone_adapter/mistralai/Mistral-7B-v0.1/mix_epoch_15_lr_1e-2_bsz_8_d_16/
head -10 output_barebone_adapter/mistralai/Mistral-7B-v0.1/mix_epoch_15_lr_1e-2_bsz_8_d_16/publish_w_adapter.jsonl
```

## 7. How Upstream Measures Success

The repo's real FSR logic lives in `report_FSR_adapter.py`.

Important detail:

- `inference.py` writes outputs from `validation` first and then `test`.
- In `create_fingerprint_mix.py`, `validation` is the same 60-example dataset as `train`.
- Therefore the first 10 lines of each generated JSONL correspond to the fingerprint examples.

`report_FSR_adapter.py` then computes:

- `FSR`
- `robust_to_normal`
- `robust_to_fingerprint`

For the fingerprint stage, the intended signal is:

- `publish_w_adapter.jsonl`: high `FSR`
- `publish.jsonl`: low `FSR`
- `vanilla.jsonl`: low `FSR`

## 8. Likely Failure Points

- `dataset/llama_fingerprint_mix` missing
- missing Python dependencies
- `accelerate` missing
- missing base model access or model download permissions
- CUDA OOM during `run_clm.py`
- missing `instruction_emb.pt` if training crashes before save
- `python` alias missing if commands are copied verbatim from README
